# نظام إدارة المطعم (RMS) - Laravel 11 & React

نظام متكامل لإدارة عمليات المطعم، يشمل لوحة تحكم إدارية، واجهة نادل، نظام POS، وشاشة مطبخ (KDS).

## المميزات الرئيسية

### لوحة تحكم المدير
- **لوحة المعلومات:** إحصائيات فورية (الفروع، طلبات اليوم، العناصر المتاحة، العملاء) مع روابط سريعة.
- **إدارة الفروع:** إنشاء وتعديل وحذف الفروع (الاسم، العنوان، الهاتف، فرع رئيسي/فرعي).
- **إدارة القاعات والأقسام:** CRUD كامل مرتبط بالفروع بعرض بطاقات.
- **إدارة الطاولات:** CRUD كامل مع نطاق السعة، الشكل (مستطيل، مربع، دائري، بيضاوي)، الموقع، وشارات الحالة.
- **إدارة التصنيفات:** CRUD كامل مع التحكم في ترتيب العرض.
- **إدارة قائمة الطعام:** CRUD كامل — رفع الصور، السعر، وقت التحضير، الوصف، الحالة (متاح/نفذ)، تبديل علامة الإضافة (Addon).
- **إدارة الطلبات:** جدول مُقسَّم بالصفحات مع بحث فوري، فلترة متعددة (الحالة، النوع)، أعمدة قابلة للترتيب، وشارات حالة الطلب والدفع.
- **تفاصيل الطلب:** تفصيل العناصر والإضافات والخصومات وسجلات الدفع، ملاحظات قابلة للتعديل (عامة وخاصة)، وسجل أحداث زمني كامل.
- **إدارة العملاء:** قائمة بعدد الطلبات، إجمالي الإنفاق، رصيد المحفظة، وتاريخ آخر طلب.
- **الإعدادات:** بيانات المطعم، اختيار العملة (أكثر من 10 خيارات)، ساعات العمل، وإدارة طرق الدفع (إضافة/تعديل/حذف مع حماية الطرق الافتراضية).

### نظام نقطة البيع (POS)
- **عرض الطاولات:** بطاقات مُجمَّعة حسب القاعة مع حالة مباشرة (متاحة/مشغولة/محجوزة) ومعلومات الطلب النشط.
- **فلترة الحالة:** عرض الكل / المتاحة / المشغولة / المحجوزة مع العدد.
- **قائمة انتظار الطلبات:** شبكة بطاقات لجميع الطلبات النشطة مع أزرار التعديل والدفع.
- **إنشاء طلب جديد:** طلبات تيك أواي أو توصيل من خلال نافذة منبثقة.
- **شاشة إدارة الطلب:** شريط جانبي للتصنيفات، شبكة عناصر مع بحث، عربة تسوق جانبية، أدوات التحكم بالكميات، ملاحظات لكل عنصر، نافذة تعديل الإضافات، وإرسال الطلب للمطبخ بنقرة واحدة.
- **شاشة الدفع (Checkout):** بحث العملاء وإنشاؤهم، حقل الخصم، دفع مرن بطرق متعددة، خصم من محفظة العميل، إضافة الفائض للمحفظة، ملاحظات عامة وخاصة، معاينة الفاتورة (تتحدث تلقائيًا)، وخيار الطباعة.
- **إشعارات الطلبات الجاهزة:** نغمة صوتية + إشعار المتصفح + تنبيه داخل التطبيق يُغلق تلقائيًا.

### شاشة المطبخ (KDS)
- بطاقات الطلبات مع مؤقتات مباشرة تتغير ألوانها (أخضر ← برتقالي ← أحمر) مع تقدم الوقت.
- شارة "طلب جديد" و"جارٍ التحضير" لكل بطاقة.
- عرض الإضافات والملاحظات الخاصة لكل عنصر.
- أزرار "بدء التحضير" و"جاهز" بمساحة لمس واسعة.
- شريط جانبي يُلخِّص إجمالي العناصر والإضافات المطلوبة.
- ساعة حائط مباشرة، تحديث تلقائي كل 30 ثانية، وتأثيرات انتقال سلسة.

### إدارة الطلبات
- **أنواع الطلبات:** تناول في المطعم، تيك أواي، توصيل.
- **حالات الطلب:** معلق ← قيد التحضير ← جاهز ← مكتمل / ملغي.
- **حالات الدفع:** غير مدفوع ← مدفوع جزئيًا ← مدفوع.
- دعم طرق دفع متعددة لكل طلب وسجل أحداث زمني كامل.

### إدارة العملاء
- إنشاء العميل مباشرةً أثناء الدفع أو البحث بالاسم/الهاتف.
- نظام محفظة/رصيد مع تتبع المدفوعات الزائدة.
- سجل الطلبات مع الإحصائيات.

### واجهة المستخدم
- واجهة عربية كاملة بالاتجاه من اليمين لليسار (RTL).
- تصميم متجاوب يدعم اللمس (Touch-friendly) على الأجهزة اللوحية وشاشات المطبخ.
- ألوان هوية المطعم: `#ee1d23` (أحمر رئيسي)، `#6f272a` (أحمر داكن)، `#feca0b` (أصفر مميز).
- مكتبتا تصميم منفصلتان: shadcn/ui للوحة الإدارة، وـ Tailwind مخصص لـ POS والمطبخ.

## متطلبات التشغيل
- PHP >= 8.2
- MySQL (مثال: MAMP أو XAMPP)
- Node.js & npm
- Composer

## خطوات التثبيت اليدوي (Manual Installation)

### 1. إعداد المشروع
قم بفك ضغط المشروع في مجلد السيرفر المحلي، ثم نفذ الأوامر التالية:

```bash
# تثبيت اعتمادات PHP
composer install

# تثبيت اعتمادات Javascript و UI
npm install
```

### 2. إعداد قاعدة البيانات
1. قم بإنشاء قاعدة بيانات جديدة باسم `restaurant_system_db` في MySQL.
2. قم بنسخ ملف البيئة وضبط بيانات الاتصال:

```bash
cp .env.example .env
```

قم بتعديل القيم التالية في ملف `.env`:
```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=8889  # (أو 3306 حسب إعداداتك)
DB_DATABASE=restaurant_system_db
DB_USERNAME=root
DB_PASSWORD=root # (أو فارغ)
```

### 3. تشغيل التهجيرات (Migrations)
```bash
php artisan key:generate
php artisan migrate
```

### 4. بناء ملفات الواجهة (Production Build)
```bash
npm run build
```

### 5. تشغيل النظام
```bash
php artisan serve
```

يمكنك الآن الوصول للنظام عبر الرابط: `http://127.0.0.1:8000`
